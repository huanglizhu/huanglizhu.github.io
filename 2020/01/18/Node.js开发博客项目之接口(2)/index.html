返回假数据：将路由和数据处理分离，以符合设计原则

思路

我们先不管接口内容，去把各个接口跑通。
blog.js和user.js中存放的是路由组件，app.js中存放服务端处理的逻辑代码，在blog.js和user.js中通过判断以后返回的是对象，所以在app.js中得到的blogData也是一个对象，但res.end返回的需要是字符串，所以需要通过JSON.stringify（）来转换一下，此时注意需要return来结束。
注意：

  1. 路由组件（函数）写好后需要通过module. exports输出出去，然后再在需要调用的js文件顶部通过require调用后才能使用。(app.js中的serverHandle不是路由组件，但是注意，函数都需要输出！)
  2. 千万不要忘记在终端执行npm run dev！！(否则就不会去监听8000端口)
  3. 设置接口路径时前面前往不要忘记api前面的\!
