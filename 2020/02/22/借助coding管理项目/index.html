借助coding管理项目

  • 更多命令可参考官方帮助文档
  • 例子：开发者a开发了登录功能的分支，开发者b开发了注册功能的分支，管理者c在master分支下使用git fetch下载所有分支并依次合并各分支功能

在coding创建一个项目

  • 我们做的每个Git项目中都需要一个“.gitignore”文件，这个文件的作用就是告诉Git哪些文件不需要添加到版本管理中。比如我们项目中的npm包(node_modules)，它在我们项目中是很重要的，但是它占的内存也是很大的，所以一般我们用Git管理的时候是不需要添加npm包的。
  • 注意：如果是先创建coding项目，那么此时需要在本地上git clone 项目地址将其下载在本地。（如果项目地址使用SSH，则需要将公钥添加到coding-个人设置-SSH公钥）

首次提交项目到coding

（如果将coding项目克隆到了本地，则忽略此步骤）

  1. 在存放需要push到coding上的文件夹中单击右键-git bash here
  2. 创建本地git仓库git init
    • 此步操作完成后，会在此文件夹下生成一个隐藏的.git后缀文件
  3. 添加文件到暂存区：git add .
  4. 提交文件到本地仓库：git commit -m "init project"
    • git commit –m “项目描述”（具体的描述方法见下方）
  5. 将本地文件推送到coding服务器:
    1. git remote add origin https://e.coding.net/hlzzz/weibo.git（origin后的是项目地址）
    2. git push origin master（master也可以是其他分支）

修改项目后提交

  1. 查看当前修改过/新增的文件:git status
  2. 查看被修改的内容:git diff
  3. 添加文件到暂存区：git add .
  4. 提交文件到本地仓库：git commit -m "相关描述"
    • 具体的描述方法见下方
  5. 将本地文件推送到coding服务器:git push origin master（master也可以是其他分支）

commit的不同项目描述的前缀

  • 我们应当按照规范将每次上传做的修改进行描述，不同的修改使用的描述前缀也是不同的。
  • 调整项目结构用refactor,比如：git commit -m "refactor:调整目录结构"
  • 添加/修改功能用feat（即feature 功能），比如：git commit -m "feat:演示路由"
  • 修改bug：git commit -m "fix:修改的bug描述"

增加分支

git checkout -b 分支名

合并分支

  1. git fetch如果本地没有全部分支，则使用fetch下载所有分支
  2. git merge 待合并分支名将待合并分支合并到当前分支中
  3. git push origin master将文件推送到服务端（master也可以是其他分支）

暂存修改

  • 如果在创建分支前进行了修改，但这个修改不应属于当前分支，则可用git stash暂存修改，然后新建/切换分支，到相应的分支下用git stash pop将修改放出
