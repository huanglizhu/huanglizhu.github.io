百度前端技术学院 第五天学习笔记（CSS flexbox）

十月 25, 2019 百度前端技术学院 本文总阅读量次

flexbox弹性盒子

  • flexbox英文参考
  • flexbox中文参考
  • flex生效需定义其父元素的display属性为flex或inline-flex
  • 父级container
  • 子项item
  • 主轴main axis（主轴是可变的，主要取决于flex-direction属性）
  • 垂直的交叉轴 cross axis

父级container的属性

display

display属性定义一个flex容器，内联或者根据指定的值，来作用于下面的子类容器。

  • 请注意，CSS列对flex容器无效。
  • float, clear, vertical-align 在flex项目中不起作用
属性值 含义
flex 块，将对象作为弹性伸缩盒显示
inline-flex 内联，将对象作为内联块级弹性伸缩盒显示

flex-direction

Flexbox是单向布局概念。可以将弹性项目想像为主要以水平行或垂直列布置。

属性值 含义
row（默认值） 从左到右ltr；从右到左rtl
row-reverse 从右到左ltr; 从左到右rtl
column 与row相似，只是变成了从上到下
column-reverse 与row-reverse相似，只是变成了从下到上

flex-wrap

默认情况下，所有弹性项目都将尝试放入一行。您可以对此进行更改，并允许该属性根据需要包装项目。

属性值 含义
nowrap （默认） 所有弹性项目都在一行上
wrap 弹性项目将从上到下分布在多行上。
wrap-reverse 弹性项目将从下到上分布在多行上。

flex-flow

  • 这是flex-direction和flex-wrap属性的简写，它们一起定义了flexbox的主轴和交叉轴。
  • 默认值为row nowrap。

justify-content

定义了沿主轴的对齐方式。(主轴是可变的，主要取决于flex-direction属性值)

属性值 含义
flex-start （默认） 从flex-direction的开始处开始放置子项
flex-end 从flex-direction的末端开始放置子项
start 子项从书写模式方向的起点开始放置
end 子项从书写模式方向的末端开始放置
left 子项从容器的左边缘开始放置，除非这与flex-direction不一致，否则它的行为类似于start
right 子项从容器的右边缘开始放置，除非这与flex-direction不一致，否则它的行为类似于start
center 子项沿线居中
space-between 子项在行中均匀分布，第一项位于开始处，最后一项位于结束处
space-around 项目在行中均匀分布，并且周围有相等的空间。请注意，从视觉上看，空间是不相等的，因为所有项目的两侧都具有相等的空间。第一项相对于容器边缘有一个单位的空间，但是它与下一项之间有两个单位的空间。
space-evenly （evenly：均匀地）分配项目，任意两个项目之间的间距（以及到边缘的间距）相等。
  • 请注意，浏览器对这些值的支持是有细微差别的。例如，space-between从来没有获得Edge支持，并且Chrome没有start/end/left/right。MDN 有详细的图表。最安全的值是flex-start，flex-end和center。
  • 还可以这些值与将两个其他关键字（safe和unsafe）配对。使用safe确保无论执行哪种类型的定位，都不能以无法滚动内容的方式推送元素以使其呈现屏幕外（例如，从顶部移出）（称为“数据丢失”） 。

align-items

用来设置每个flex元素在侧轴上的默认对齐方式

属性值 含义
stretch （默认） 拉伸以填充容器（仍然遵守最小宽度/最大宽度）
flex-start/ start/ self-start 项目放置在cross-axis的起点,它们之间的区别是微妙的，并且是遵守flex-direction规则或writing-mode规则的
flex-end/ end/ self-end 项目放置在cross-axis的末端。差异再次是微妙的，尊重flex-direction规则与writing-mode规则
center 项目在cross-axis上居中
baseline 项目对齐，例如基线对齐

可以结合safe和unsafe关键字与上面的属性值一起使用（但是注意浏览器支持），他可以防止内容变得不可访问。

align-content

当在横轴上有多余的空间时，这会将弹性容器的线对齐在内部，这与在主轴justify-content内对齐单个项目的方式类似。
注意：只有一行flexbox时，此属性无效。

属性值 含义
stretch （默认） 线拉伸以占据剩余空间
flex-start/ start 子项从容器开始位置开始放置。flex-start（受更多支持）支持flex-direction而start支持writing-mode方向。
flex-end/ end 子项从容器末端开始放置。flex-start（受更多支持）支持flex-direction而start支持writing-mode方向。
center 子项位于容器中心
space-between 子项平均分配，第一行在容器的开头，而最后一行在容器的结尾
space-around 项目均匀分布，每一行周围具有相等的空间（注意：两行之间有两个单位的空间，参考justify-content属性的space-around属性值）
space-evenly 项目均匀分布，从视觉上看，每一个行周围具有相等的空间

区别justify-content属性与align-content属性

  • justify-content属性是对齐一行内的子项。
  • align-content属性是对齐多行内的不同行。

子项item的属性

order顺序

默认情况下，flexbox按源顺序排列。但是，该order属性控制子项在flex容器中出现的顺序。

属性值 含义
<整数> 默认为0

flex-grow弹性增长

  • 定义了flexbox在必要时增长的能力。它决定了子项在父级容器（flex container）内部占用多少可用空间。
  • 如果所有项目flex-grow都设置为1，则容器中的剩余空间将平均分配给所有子项。如果其中一个孩子的值为2，则剩余空间将占其他孩子的两倍（或者至少会尝试）。

    属性值不能为负数。

flex-shrink弹性收缩

定义了flexbox在必要时收缩的能力。

属性值不能为负数。

flex-basis弹性基础

  • 定义了剩余空间分配之前元素的默认大小。
属性值 含义
长度 例如20％，5rem等
关键字 auto：查看我的width与height属性(这是由main-size关键字临时完成的，直到弃用)
content(此关键字不能很好地支持,不建议使用):根据子项的内容调整大小

  • 属性值如果设置为0，则不考虑内容周围的额外空间。
  • 属性值如果设置为auto，则根据其flex-grow值分配额外的空间。

flex简写flex-grow、flex-shrink和flex-basis

  • 建议使用该简写属性！
  • 这是flex-grow、flex-shrink和flex-basis组合的简写。
  • 第二和第三个参数(flex-shrink和flex-basis)是可选的。
  • 默认值是0 1 auto。

align-self自我调整

该属性为单个flex项目覆盖默认的对齐(或由align-items指定的对齐)

注意：float（浮动）, clear（清除浮动） 和 vertical-align对flex子项没有影响。
